Description

This week we will launch our semester-long development project. The project will evolve over the duration of the course. Each week's assignment is designed to highlight a particular phase of the Software Development Life Cycle (SDLC).

Although most engineers agree that there will be some inherent recursion and overlap between the phases of the SDLC, and that the process framework will be tailored according to the scope of each project, the sequence and objective of each phase shouldn't be overlooked.

This week you'll be identifying and researching project ideas. In the next couple of weeks, you'll narrow the scope and present your idea and preliminary design to your peers, gather feedback, and further refine your project plan.

 

Your project doesn't need to be brand new and unique, but your implementation will.

Begin to think about the kind of project to showcase. List the specific functionalities of the chosen kind of project, and some preliminary investigation toward implementing the project. All projects must include:

  • a data component
    • The data will be stored in a database
    • Data access will be via an intermediate API
  • and an interface component
    • There may be multiple applications accessing the API for external users and administration
    • Applications may be web, native mobile apps, console apps, or platform-specific native applications.
There is a lot of detail and room for interpretation here. Now is the time to ask questions. Discord is a good place to discuss your questions.

Background

This project was intended for developing cloud-hosted web applications.

It is still my recommendation that you consider developing your project for this platform, but many students will have compelling reasons for developing for another purpose.

Your project will include a database, accessed through an API, and a front-end GUI. This is well-suited for a web app, hosted locally or on another server or servers. We will be discussing your approach over the next couple of weeks, so now is a good time to consider options.

For this project, start by browsing the internet and considering websites that you commonly visit. What is it about the site that makes it useful? Are there other sites that enable similar functionality? What makes it appealing? What problem does it solve?

    The project is designed to:
  • give a realistic look at the larger life cycle of taking ideas and turning them into useful software-based systems
  • provide insight into the organizational and inter-personal aspects of project management
  • give experience with change management, builds and controls, and other management disciplines necessary to keep projects heading towards success (and keep costs within budget!)
  • give design, build, test and support experience with software systems

This is an individual project. Each student is building a portfolio that reflects their growth and development as a software and systems professional.

It is also a collaborative project. There will be opportunity to discuss and work with classmates informally and formally as parts of task-focused teams.

If two or more students have closely aligned projects, a shared database solution may also be acceptable. If you want to take this approach, please discuss with me. Database schema can be developed and managed jointly, and data populated individually but the front-end applications should still be unique.

Some of these deliverables will be formal documents, or code files, or reports; others will be accomplished with formal or informal presentations, discussions, whiteboarding, or other options.

Projects will be hosted on public Github repositories. The project owner will maintain all control of the code and artifacts, but these will have public read-only visibility.

The instructor will guide the project and the process, step by step.

Web Page
Provides visual and text content that the user can see and read, but not affect in any way.
Web Application
A network-hosted program which enables the user to not only read the page content but also manipulate the data on this page.
User- and data-interaction are supported by common network protocols including but not limited to http, https, ftp, and others.
Mobile App
Applications written specifically to target mobile devices, usually iOS or Android. Supporting multiple (both) platforms usually requires writing separate applications.

 

Sample Ideas

You may choose your own project. Here is a list of sample projects for you to get started.

This list is intended to spark ideas, not limit them. You may choose one of these or something else entirely.

School system
This may consist of a student module, an instructor module, a curriculum module, and provides for the assignment of instructors to classes, students to classes, course management, student management, an d instructor management. As example would be the California State University (CSU) student information system.
Electronic learning system
This may consist of a collection of courses. Each course may allow for tests, quizzes, exercises, research papers, discussions, etc. As example would be Cougarview.
An auction system.
This my consist of a sellers module, a bidders module, a purchase module, and provide for different kinds of sales (i.e., open bidding, bidding with reserve, buy now, etc.). It would include user registrations and appropriate databases of auctions, transactions, etc. An example would be eBay.
Messaging system
This would include user registrations, posting messages, sending messages, composing groups, and similar functionality. An example would be Twitter.
Encyclopedia
This would include editor registrations, posting entries, editing entries, search functions, analytical functions, and similar functionality. An example would be Wikipedia.
Electronic voting system
This would include voter registrations, candidate qualification, casting ballots with appropriate authentication, and calculation of results. An example would be the current voting system in Georgia.
Networking system
This would include user registrations, posting employment data, school data, certification data, allow the posting of jobs, etc. An example would be LinkedIn.
Online store
This would implement the display of merchandise, a shopping cart, a payment module, and inventory control mechanism, etc. An example would be Amazon.
Database GUI
This would implement a graphical front end to a database. Modules would (1) create or drop databases, (2) create, drop, and alter tables, (3) insert, update, or delete data, and (4) run simple queries. An example would be TOAD.
Organization system
This might include member registration, a member directory, an event calender, a newsletter, a photo album, a FAQ, or other functions suitable for organizations. Examples would include a social organization, a service organization, a band or orchestra, a church, an athletic team, etc.
Automated teller machine
This might include customer authentication, checking account status, acceptance of deposits, and dispensing cash.
A strategy game
At a minimum, this would consist of a user interface, a data source, and a logic engine implementing the game rules.

Deliverable

The deliverable is a two (2) to three (3) page discussion (.doc or .pdf) of three (3) or four (4) specific proposed project ideas, and a short description of the characteristics of each.

The description of each proposed project should contain enough information so that readers can understand the nature of the project. This may also include examples of similar, existing implementations.

 

Your papers should consist of a short introduction, a short conclusion, and a body containing the substance of your discussion.

Your paper will have a title, author, and date. The subdivisions of your paper should have appropriate section and subsection headings.
